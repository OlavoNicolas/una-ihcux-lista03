# 1ª Heurística de Nielsen: Visibilidade do Status do Sistema

## Descrição

A primeira heurística de usabilidade de Jakob Nielsen estabelece que o sistema deve sempre manter os usuários informados sobre o que está acontecendo, por meio de feedback apropriado e em tempo razoável.

## Objetivo

Garantir que o usuário nunca fique em dúvida sobre o estado atual do sistema, aumentando a confiança e reduzindo a ansiedade durante a interação.

## Boas práticas

* Exibir indicadores de carregamento (spinners, barras de progresso)
* Fornecer feedback imediato após ações do usuário (cliques, envios de formulário)
* Informar claramente estados como "processando", "salvo", "erro" ou "concluído"
* Atualizar dinamicamente o status em operações longas

## Problemas comuns

* Falta de feedback após uma ação
* Tempos de espera sem indicação visual
* Mensagens vagas ou inexistentes sobre erros ou progresso

## Exemplos

* Barra de progresso ao fazer upload de arquivos
* Mensagem "Salvando..." seguida de "Salvo com sucesso"
* Indicador de carregamento ao abrir uma página

## Benefícios

* Melhora a experiência do usuário (UX)
* Reduz incerteza e frustração
* Aumenta a percepção de desempenho do sistema

## Conclusão

A visibilidade do status do sistema é essencial para uma interação clara e eficiente. Sempre que o usuário realizar uma ação, o sistema deve responder de forma visível, rápida e compreensível.
